# Schedule / Calendar — Architecture & Usage

## Overview

The Schedule tab shows all fixtures across leagues in two views: a **fixtures
list** (grouped by matchday) and a **calendar** (month grid). Fixtures are
generated on the Rust backend at game start (round-robin, playoffs, friendlies)
and travel inside `league.fixtures[]` as part of the `Game` state. The calendar
view additionally merges scrim slots from the current week.

---

## Data flow

```
  ├─ generate_preseason_friendlies()      (matchday=0, weeks before start)

        │
        ├──────────────────────────────────────────┐

  ScheduleTab (legacy)                      ScheduleTabV2 (v2)

        ├── view="fixtures"                        ├── view="fixtures"
```

## Fixture types

| `match_type` | Generated by | Matchday range |
|---|---|---|
| `Friendly` | `generate_preseason_friendlies()` | 0 |
| `PreseasonTournament` | `generate_schedule_from_config()` | 0 |
| `League` | `generate_single_round_league*()` | 1..n |
| `Playoffs` | `generate_winter/spring_summer_playoffs()` | n+1.. |

Each `Fixture` has:

| Field | Type | Description |
|---|---|---|
| `id` | `String` | Unique ID |
| `matchday` | `u32` | Round number (0 = preseason) |
| `date` | `String` | `"YYYY-MM-DD"` |
| `home_team_id` | `String` | Home side |
| `away_team_id` | `String` | Away side |
| `match_type` | `MatchType` | League / Friendly / PreseasonTournament / Playoffs |
| `best_of` | `u8` | 1 (default), 3, or 5 |
| `status` | `FixtureStatus` | Scheduled / InProgress / Completed |
| `result` | `Option<MatchResult>` | Score + report when played |

## Fixture generation (Rust)

All logic in `src-tauri/crates/olm_core/src/schedule.rs`.

### Round-robin (league)

Uses the **circle method** (`round_robin_pairings` in `turn/mod.rs:457`):

1. Fix index 0 in place
2. Rotate all other indices by one position each round
3. Generate `n-1` rounds for single round-robin
4. Double round-robin = single + return leg (home/away reversed)

Handles odd team counts with a "bye" sentinel.

### Date spacing

League fixtures default to **7 days apart**. The `ScheduleConfig` manifest can
specify `superweek_offsets` for condensed schedules (e.g. LEC superweeks with
shorter gaps).

### Playoffs

- **Winter**: top 8, double elimination bracket
- **Spring/Summer**: top 6, double elimination bracket
- Generated via `generate_winter_playoffs()` / `generate_spring_summer_playoffs()`

### Preseason friendlies

Generated by `generate_preseason_friendlies()` — matchday 0, spread across
weeks before the season start.

```
┌──────────────────────────────────────────┐
│  [Partidos]  [Calendario]                │  ← segmented control
├──────────────────────────────────────────┤
│  ┌── Esta semana ───────────────────┐    │
│  │  MON  TUE  WED  THU  FRI  SAT  SUN │  ← WeekScheduleCard (v2 only)
│  │   10   11   12   13   14   15   16  │
│  │              vs                     │
│  │             G2                      │
│  └────────────────────────────────────┘
│                                          │
│  ┌─ Matchday 3 — 2026-02-15 ──────────┐ │
│  │ BO1  G2  2 - 1  FNC  ›             │ │
│  │ BO1  KOI    vs    VIT               │ │
│  │ BO1  TH   1 - 2  BDS  ›             │ │
│  └────────────────────────────────────┘ │
│  ┌─ Matchday 4 — 2026-02-22 ──────────┐ │
│  │ ...                                 │ │
│  └────────────────────────────────────┘ │
└──────────────────────────────────────────┘
```

**Logic** (both legacy and v2):

1. Group fixtures by `getFixtureGroupKey()`:
   - `league-{matchday}` for League
   - `playoffs-{matchday}` for Playoffs
   - `{match_type}-{date}` for Friendlies / PreseasonTournament
2. Sort groups by `date` then `matchday`
3. Each group renders as a Card with:
   - Header: group label (e.g. "Matchday 3 — 2026-02-15")
   - Rows: BO badge, home team, score / "VS", away team, chevron for results
4. Completed fixtures with stored draft results open `<DraftResultScreen>`
5. Playoffs get `<PlayoffBracketBoard>` above the list

**Color coding** per row:
- User won: `bg-emerald-500/5` (v2) / blue tint (legacy)
- User lost: `bg-red-500/5` (v2) / red tint (legacy)
- User pending: `bg-primary/5`
- Non-user: no background

**Component**: `ScheduleCalendarView` (shared between v2 and legacy, 589 lines).

Props:
```ts

  gameState: GameStateData;

  competitionLabelMap: Map<string, string>;   // fixture.id → league name
```

**Layout:**
- Header: `◀` / month label / `▶` + "Today" button
- Weekday row: localized day names
- 6-week grid (42 cells)
- Each cell shows: day number (highlighted if today), up to 3 event chips,
  season start indicator (`🏁` Flag), playoffs estimate (`🏆` Trophy),
  "+N more" link if overflow

**Event chips:**
- `FixtureChip`: team logos (compact: icons only; full: names + scores),
  BO badge, competition label. Color-coded by user team result.
- `ScrimChip`: amber tag with `Swords` icon, opponent name

**Day overlay (modal):** clicking "+N more" opens a centered modal with all
events for that day in full detail.

**Calendar-specific metadata:**
- **Season start**: first league matchday's date, shown as Flag icon
- **Playoffs estimate**: 7 days after last league matchday, shown as Trophy
- **Scrims**: merged from `useScrimContextWithFallback()` → `buildSelectedScrimEvents()`

## Best-of inference

`inferBestOf()` resolves the series length for each fixture:

| Condition | Result |
|---|---|
| `fixture.best_of` explicitly set | That value |
| Playoff fixture within known bounds | 5 (if close to final) or 3 |
| Friendly with known series length | Series length |
| Everything else | 1 (single map) |

---

## Score normalization

`normalizeLolScore()` converts raw match results into displayable scores,
accounting for:

- **Completed fixtures**: reads from `fixture.result`
- **Draft results**: reads from `localStorage` (stored during live match when
  the user controlled draft but match is not yet marked Completed)
- **BO normalization**: BO3 winner with 2-0 shows `{home: 2, away: 0}`

## Gotchas

- **Calendar is desktop-only**: hidden below `md` breakpoint. On mobile the
  segmented control hides the calendar option entirely.
- **All leagues merged in calendar**: `view === "calendar"` passes
  `allFixtures` (all leagues flattened), while fixtures view shows only the
  active league.
- **Scrims only in calendar**: scrim slots do not appear in the fixtures list,
  only as amber chips in the calendar grid.
- **Playoffs estimate**: the calendar computes an estimated playoff start date
  (7 days after last league matchday). This is a heuristic, not a guarantee.
- **Draft result localStorage**: `readStoredFixtureDraftResult()` checks
  `localStorage` for draft snapshots. These are set during live match and
  cleared when the fixture is officially completed.
- **BestOf for friendlies**: `buildBestOfContext()` groups back-to-back friendlies
  between the same teams into a series. A single friendly is BO1.
